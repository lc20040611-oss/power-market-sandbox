# 中国电力市场规则沙盒平台

面向电力市场规则、机制设计、交易仿真、实验管理和论文研究的研究实验平台。

## 项目介绍

该项目基于 `Next.js + TypeScript + Tailwind CSS + Recharts` 构建，当前版本支持：
- 多时段市场出清与储能时序优化
- 规则参数配置与多场景对比
- 中长期合约与现货偏差结算
- 新能源消纳与弃风弃光分析
- 储能参与市场分析
- 容量补偿机制分析
- 市场力分析
- 实验管理与参数扫描
- CSV / JSON 导出
- 论文图表与研究报告草稿生成

## 功能列表

- `Dashboard`：平台概览与模块导航
- `规则库`：市场机制卡片与规则说明
- `规则配置`：统一出清价、按报价支付、偏差考核、新能源优先、容量补偿等规则参数
- `市场仿真`：主体输入、规则驱动出清、结果表和图表
- `结果分析`：最近一次仿真结果和规则模板对比
- `中长期合约`：合约收益、偏差电量、偏差结算和合约覆盖率
- `新能源消纳`：消纳率、弃风弃光量、补贴收益
- `储能参与`：SOC、充放电功率、套利收益
- `容量机制`：容量补偿价格与系统可靠性变化
- `市场力分析`：HHI、CR3、CR5、RSI、策略报价影响
- `实验管理`：实验模板、参数扫描、实验记录、结果表和图表
- `论文工具`：论文图表、自动分析文字、Markdown 报告草稿

## 部署方式

项目可直接部署到 Vercel。

### 推荐步骤

1. 将项目推送到 Git 仓库
2. 在 Vercel 中导入仓库
3. 使用默认 Next.js 构建配置
4. Build Command：

```bash
npm run build
```

5. Install Command：

```bash
npm install
```

当前项目默认可无环境变量运行，并已支持本地 SQLite 实验归档。

## 本地存储策略

- `localStorage`：
  - 最近市场输入
  - 最近规则配置
  - 最近实验摘要
  - 当前选中的实验 ID
- `sessionStorage`：
  - 当前会话的完整实验结果

这样可以避免在浏览器长期保存大规模参数扫描结果。

## 当前模型简化假设

- 多时段逐时出清，暂未实现跨日安全约束联立优化
- 不含真实网络约束、节点电价和潮流约束
- 中长期合约结算采用简化财务结算
- 偏差考核采用简化的申报量与执行量偏差
- 储能采用启发式峰谷套利策略
- 容量机制采用简化容量价格与考核惩罚
- 市场力分析采用集中度和策略扰动近似
- 实验归档默认采用本地 SQLite，未接入远程共享数据库
